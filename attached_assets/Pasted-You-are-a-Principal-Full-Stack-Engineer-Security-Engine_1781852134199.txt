CRITICAL FIXES
────────────────────────────────────────

1) Protect booking routes

- Add authentication middleware
- Enforce ownership checks
- Allow admin override
- Prevent IDOR access
- Return proper 401/403 responses

2) Secure guest PII
- Do not expose phone, email, address, or ID details to unauthorized users
- Only owner or admin can view full booking data
- Return sanitized booking objects for public/limited access

3) Fix payment amount tampering
- Never accept amount directly from the frontend
- create-order must accept bookingId only
- Fetch booking from database
- Calculate payable amount server-side
- Ignore client-sent amount completely

4) Fix webhook security
- Razorpay webhook signature verification must be mandatory
- If webhook secret is missing, fail startup or reject requests
- Reject any webhook with invalid signature
- Do not allow fake payment confirmations

5) Sync webhook with bookings
- When payment is captured, update the related room booking or event booking automatically
- Update payment status, booking status, and confirmation timestamps

6) Prevent double booking
- Fix race conditions in room booking creation
- Use transactions, locking, or atomic conflict-safe logic
- Prevent two users from booking the same room inventory at the same time
- Support multiple rooms per room type correctly using total inventory count
